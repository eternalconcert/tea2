API

Quick reference for the Tea language: syntax, built-ins, and core constructs. Tea is meant for small scripts and automation, not as a full “full-stack” language.

Language

Syntax

  • Statements end with a semicolon ;.
  • Blocks use curly braces { … }.
  • Strings use double quotes; escapes: \n, \t, \", \\.
  • Modules: import "path/to/file.t"; — exported symbols come from files that use export.
  • Teahouse modules: import "@vendor/module.t"; resolves to ./teahouse/vendor/module.t (from the current working directory).
  • If names clash, @... imports have lower priority and do not overwrite existing symbols.

Keywords

Control flow: if, else, for, while, return, fn, break, continue, import, export, throw, not. Booleans: true, false.

Identifiers

Start with a letter or underscore, then letters, digits, or underscores: [A-Za-z_][A-Za-z0-9_]*. Names are case-sensitive.

Literals

  • Integers: 42, -1
  • Floating-point: 3.14, -0.5
  • Strings: "hello"
  • Booleans: true, false
  • Arrays: [1, 2, "a"] — elements may mix types.
  • Dictionaries: {"key": value} — keys are strings; values may be any Tea expression (including nested dicts/arrays).

Comments

  • Line comment: // to end of line
  • Shebang line is ignored: #! …
  • Block comment: /* … */

Types

Explicit types (lowercase, as in the lexer): int, float, str, bool, void, array, dict. Function parameters can also use fn when a function value is expected. 

int n = 10;
str s = "text";
bool ok = true;
array xs = [1, 2, 3];
dict d = {name: "tea", year: 2026};

Variables and functions

Declare with a type and optional initializer; functions use fn, return type before the name, or void when there is no return value. Variadic functions use ...name. 

int x;
x = 1;

int fn add(int a, int b) {
    return a + b;
};

void fn greet(str name) {
    print("Hello, " + name);
};

void fn log(...parts) {
    print(parts);
};

Function values and predicates

Functions can be passed as values to parameters declared with fn. This is useful for helpers such as filter from common/iterable.t. Prefix a predicate with not to pass the inverse predicate without writing a wrapper function. 

import "common/iterable.t";

bool fn isNotEmptyString(str value) {
    return len(value) > 0;
};

array lines = split(cmd("printf 'a\n\nb\n'"), "\n");
array nonEmpty = filter(lines, isNotEmptyString);
array empty = filter(lines, not isNotEmptyString);

Operators

  • Arithmetic: + - * / %
  • Comparison: == != < <= > >=
  • Logic: not, and (also && / &), or (also || / |)

Control flow

Conditionals

if (condition) {
    // ...
} else {
    // ...
};
if (not done) {
    print("still running");
};

Loops

for (int i = 0; i < len(xs); i = i + 1) {
    if (i == 10) {
        break;
    };
    print(xs[i]);
};

while (n > 0) {
    n = n - 1;
};

Indexing

Arrays and strings use integer indexes; dictionaries use keys. Indexing can be nested. 

array xs = ["tea", "cup"];
dict page = {title: "Home", tags: xs};

print(xs[0]);              // tea
print(page["tags"][1]);    // cup
print("tea"[0]);           // t

Built-ins and I/O

The runtime provides, among others:

  • print(…) — write to stdout
  • stdout("text") — write without a trailing newline (e.g. prompts)
  • input — read one line from stdin (typical: str name = input;)
  • cmd("shell command") — run a shell command, output as a string
  • env("NAME") — read an environment variable, or return an empty string if it is not set
  • read("path") / write("path", content) — read/write files
  • len(x) — length (string, array, or dict)
  • split(str, delimiter), find(str, pattern) — string helpers
  • regexTest, regexMatch, regexFind, regexCapture, regexCaptureAll — lightweight regular expression helpers
  • dictKeys(dict) — returns an array of keys (as strings)
  • dictValues(dict) — returns an array of values
  • json(str) — parse JSON strings; objects become dicts, arrays become arrays, and null becomes an empty dict
  • sleep(seconds), quit(), assert(actual, expected[, "message"])
  • cast(…) — type conversion
  • SYSARGS, LRC — command-line args and last return code (see interpreter docs)

Regular expressions

Tea includes a small regex engine for common string matching. It supports anchors ^/$, character classes, negated classes, ., quantifiers */+/?, escapes such as \d, \w, \s, and capture groups. 

assert(regexTest("hello tea", "tea"), true);
assert(regexMatch("tea42", "tea\\d+"), true);
assert(regexFind("one two three", "(t\\w+)"), [4, 8]);

array first = regexCapture("name: tea, count: 42", "name:\\s*(\\w+),\\s*count:\\s*(\\d+)");
assert(first, ["tea", "42"]);

array all = regexCaptureAll("name: tea; name: cup", "name:\\s*(\\w+)");
assert(all, [["tea"], ["cup"]]);

regexFind returns start indexes. If the pattern contains capture groups, they are used for matching but ignored in the result. Use regexCapture for the first match's groups and regexCaptureAll for all matches. Each entry from regexCaptureAll is an array of captured groups, so a pattern with two groups returns values like [["tea", "42"], ["cup", "7"]].

\w only matches letters, digits, and underscores. For paths or template names that may contain slashes or dots, use a character class such as [^%]+ and trim the result if needed.

Common helpers

The repository includes small reusable modules under common/. Import them explicitly when needed.

  • common/iterable.t: dictItems(dict), arrayContains(array, str), filter(array, fn)

HTTP client

http sends plain http:// requests and returns a dictionary with status, body, headers, and error. Convenience wrappers are available as request, get, post, put, patch, and delete. 

dict response = get("http://127.0.0.1:8080/hello");
assert(response["status"], 200);

dict created = post("http://127.0.0.1:8080/items", "{\"ok\":true}");
dict payload = json(created["body"]);

dict custom = http({
    method: "PUT",
    url: "http://127.0.0.1:8080/items/1",
    headers: {"Content-Type": "application/json"},
    body: "{\"name\":\"tea\"}"
});

HTTP server

serve starts a simple HTTP server. The handler receives a request dict (method, path, query, body, headers) and returns a response dict (status, headers, body). 

dict fn app(dict req) {
    return {
        status: 200,
        headers: {"Content-Type": "text/plain"},
        body: "hello from tea"
    };
};

serve({port: 8080, handler: app});

More helpers can be pulled in via import from libraries (for example common/string.t with replace, replaceMany, and so on).
